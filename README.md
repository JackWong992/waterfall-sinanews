## 懒加载实现原理
准备工作： 找固定的图片代替需要缓存的图片

封装三个函数：
* 判断元素时候出现在用户视野内
  * 大致为：窗口的高度+元素滚动的距离 >= 元素具体页面顶部的距离
* 加载缓存的图片
  * 大致为：将固定图片的标签(src)修改为真实图片的地址
* 开始执行，先判断是否满足封装的第一个函数，满足执行第二个函数。
* 滚动触发事件：执行封装的第三个函数

注意事项：首屏是默认出现在用户视野内的，所以要先执行一次封装的第三个函数
